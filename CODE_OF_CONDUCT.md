# Code of Conduct

This project uses a simple professional conduct standard.

Participants should be respectful, evidence-led, and constructive. Do not submit
or request real client data, protected information, credentials, or material that
the contributor does not have the right to share.

Unacceptable behavior includes harassment, discriminatory language, personal
attacks, doxxing, secret exposure, or pressure to misrepresent portfolio work as
production client delivery.

Issues should be handled by the repository owner. Reports can be made through
GitHub profile contact routes or private vulnerability reporting where relevant.
